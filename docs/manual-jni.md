## Caveats

- **Heuristic, not semantic.** The classifier is regex-based. Member 
  detection covers methods, constructors, and property accessors but 
  may misattribute a JNI call inside an unusual member shape (lambda 
  bodies stored in fields, expression-bodied members spanning many 
  lines). Such calls land under a synthetic `(file-level)` row so no 
  call disappears.
- **`[ComposeBridge]` / `[ComposeFacade]` exclusion** is by attribute 
  presence on the immediately preceding lines. If a hand-written 
  member sits next to a generator-decorated one in the same file, both 
  classifications are emitted independently.
- **"Why not generated?"** is extracted from existing source comments 
  — XML doc `<remarks>` paragraphs containing _Why raw JNI_, 
  `// Why raw JNI:` lines, or adjacent `// ...` blocks. Entries that 
  show **TODO** are docs gaps in the source itself.
- **Call sites, not unique JVM symbols.** A cached `static IntPtr s_X` 
  bound once and used three times counts as three call sites.
- **Microsoft.AndroidX.Compose JCWs only.** The JCW table tracks `[Register("net/compose/…")]`, 
  the namespace this project owns. JCW shapes the Mono.Android binding 
  generator emits for upstream interfaces are not flagged.
