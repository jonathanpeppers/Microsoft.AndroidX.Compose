## Caveats

- Matching is **short-name** based. A Kotlin `Button` matches a C# `Button` 
  regardless of namespace nesting; collisions are possible but rare in Compose.
- Overloads on the Kotlin side collapse to one entry (the one with the most params).
- Modifier extensions only match if a method of the same name exists in PublicAPI; 
  the matcher doesn't check the extension's parameter shape.
- `@ExperimentalFoundationApi`, `@ExperimentalMaterial3Api`, etc. annotations are 
  preserved in symbol metadata but don't currently affect scoring.
- See `scripts/api-comparison.cs` for the full set of heuristics.
